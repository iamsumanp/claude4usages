Journey maps to domain model: Step 2 creates ProviderAccountConfig (label, email, org, avatar color). Step 3 sets probeConfig (CLI profile or API token env var). Step 4 calls refreshAccount() to verify connectivity. Step 5 adds to MultiAccountSettingsRepository and appears in parallel monitoring.
